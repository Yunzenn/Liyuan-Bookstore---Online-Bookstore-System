# 逻辑结构设计

## 1. E-R图向关系模型的转换

### 1.1 转换原则

1. **实体转换规则**：每个实体转换为一个关系（表）
2. **属性转换规则**：实体属性直接转换为关系属性
3. **关系转换规则**：
   - 1:1 关系：合并到任意一方
   - 1:N 关系：将"1"方主键加入"N"方作为外键
   - M:N 关系：新建关系，包含双方主键

### 1.2 本项目转换策略

| 关系类型 | 转换策略 | 说明 |
|----------|---------|------|
| Customer 1:N Order | Order表添加customer_id外键 | 客户一对多订单 |
| Order 1:N OrderItem | OrderItem表添加order_id外键 | 订单一对多明细 |
| Book 1:N OrderItem | OrderItem表添加ISBN外键 | 图书一对多明细 |

## 2. 关系模式定义

### 2.1 客户表 (customer)

```sql
CREATE TABLE customer (
    customer_id     INT AUTO_INCREMENT,
    customer_name   VARCHAR(50) NOT NULL,
    address         VARCHAR(200),
    phone           VARCHAR(20),
    email           VARCHAR(100) NOT NULL,
    password_hash   VARCHAR(255) NOT NULL,
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    PRIMARY KEY (customer_id),
    UNIQUE KEY uk_email (email)
);
```

**候选码分析：**
- 主候选码：customer_id
- 次候选码：email（具有唯一性约束）

**范式判断：3NF / BCNF**
- 满足1NF：所有属性都是原子值
- 满足2NF：单属性主键，不存在部分依赖
- 满足3NF：无传递依赖
- 满足BCNF：所有决定因素都是候选键

### 2.2 图书表 (book)

```sql
CREATE TABLE book (
    ISBN            VARCHAR(13),
    title           VARCHAR(200) NOT NULL,
    author          VARCHAR(100) NOT NULL,
    price           DECIMAL(10,2) NOT NULL,
    publisher       VARCHAR(100),
    publish_year    YEAR,
    stock           INT DEFAULT 0,
    
    PRIMARY KEY (ISBN),
    CHECK (price >= 0),
    CHECK (stock >= 0)
);
```

**候选码分析：**
- 主候选码：ISBN
- 无其他候选码（书名、作者可能重复）

**范式判断：3NF / BCNF**
- 满足1NF：所有属性都是原子值
- 满足2NF：单属性主键
- 满足3NF：无传递依赖
- 满足BCNF：ISBN是唯一决定因素

### 2.3 订单表 (order)

```sql
CREATE TABLE `order` (
    order_id        INT AUTO_INCREMENT,
    customer_id     INT NOT NULL,
    order_date      DATETIME DEFAULT CURRENT_TIMESTAMP,
    status          ENUM('待付款','已付款','已发货','已完成','已取消') DEFAULT '待付款',
    total_amount    DECIMAL(10,2),
    
    PRIMARY KEY (order_id),
    FOREIGN KEY (customer_id) REFERENCES customer(customer_id)
        ON DELETE RESTRICT
        ON UPDATE CASCADE
);
```

**候选码分析：**
- 主候选码：order_id
- 无其他候选码

**范式判断：3NF / BCNF**
- 满足1NF：所有属性都是原子值
- 满足2NF：单属性主键
- 满足3NF：无传递依赖
  - customer_id → customer_name 是外键依赖，不是传递依赖
  - order_id → customer_id → customer_name 中，customer_name直接依赖于customer_id，但不传递依赖于order_id（因为customer_name不在order表中）
- 满足BCNF：order_id是唯一决定因素

### 2.4 订单明细表 (order_item)

```sql
CREATE TABLE order_item (
    order_id        INT,
    ISBN            VARCHAR(13),
    quantity        INT NOT NULL,
    unit_price      DECIMAL(10,2) NOT NULL,
    
    PRIMARY KEY (order_id, ISBN),
    FOREIGN KEY (order_id) REFERENCES `order`(order_id)
        ON DELETE CASCADE
        ON UPDATE CASCADE,
    FOREIGN KEY (ISBN) REFERENCES book(ISBN)
        ON DELETE RESTRICT
        ON UPDATE CASCADE,
    CHECK (quantity > 0),
    CHECK (unit_price >= 0)
);
```

**候选码分析：**
- 主候选码：(order_id, ISBN) 联合主键
- 无其他候选码

**范式判断：**
- 满足1NF：所有属性都是原子值
- 满足2NF：quantity和unit_price都完全依赖于整个联合主键，不存在部分依赖
  - quantity 依赖于 (order_id, ISBN)，不能仅由order_id或ISBN决定
  - unit_price 依赖于 (order_id, ISBN)，同一图书在不同订单可能价格不同
- 满足3NF：无传递依赖
  - quantity和unit_price不依赖于除主键外的任何属性
- 满足BCNF：(order_id, ISBN)是唯一决定因素

## 3. 范式详细分析

### 3.1 第一范式 (1NF) 验证

**定义**：关系中的每个属性都是不可再分的原子值。

| 表名 | 验证结果 | 说明 |
|------|---------|------|
| customer | ✓ 满足 | 所有字段均为原子值 |
| book | ✓ 满足 | 所有字段均为原子值 |
| order | ✓ 满足 | 所有字段均为原子值 |
| order_item | ✓ 满足 | 所有字段均为原子值 |

**注意**：address字段如需更细粒度查询，可拆分为省、市、区、详细地址，当前设计视为单一原子值。

### 3.2 第二范式 (2NF) 验证

**定义**：在1NF基础上，非主属性完全依赖于候选码（不存在部分依赖）。

| 表名 | 验证结果 | 说明 |
|------|---------|------|
| customer | ✓ 满足 | 单属性主键，不存在部分依赖 |
| book | ✓ 满足 | 单属性主键，不存在部分依赖 |
| order | ✓ 满足 | 单属性主键，不存在部分依赖 |
| order_item | ✓ 满足 | 联合主键，所有非主属性都完全依赖于整个主键 |

**order_item详细分析：**
- 主键：(order_id, ISBN)
- 非主属性：quantity, unit_price
- quantity 完全依赖于 (order_id, ISBN)：同一图书在不同订单中数量不同
- unit_price 完全依赖于 (order_id, ISBN)：同一图书在不同订单中价格可能不同（促销、折扣等）

### 3.3 第三范式 (3NF) 验证

**定义**：在2NF基础上，非主属性不传递依赖于候选码。

| 表名 | 验证结果 | 说明 |
|------|---------|------|
| customer | ✓ 满足 | 无传递依赖 |
| book | ✓ 满足 | 无传递依赖 |
| order | ✓ 满足 | 无传递依赖 |
| order_item | ✓ 满足 | 无传递依赖 |

**潜在传递依赖分析：**

customer表：
- 假设存在：customer_id → city → zip_code
- 当前设计未存储city和zip_code，address为单一字段
- 如需拆分address，应建立单独的address表

book表：
- 假设存在：ISBN → publisher → publisher_address
- 当前设计未存储出版社地址
- 如需存储，应建立单独的publisher表

order表：
- customer_id是外键，不是传递依赖
- 订单表只存储customer_id，不冗余存储客户信息

### 3.4 BCNF验证

**定义**：在3NF基础上，所有决定因素都是候选键。

| 表名 | 决定因素 | 候选键 | 验证结果 |
|------|---------|--------|---------|
| customer | customer_id, email | customer_id, email | ✓ BCNF |
| book | ISBN | ISBN | ✓ BCNF |
| order | order_id | order_id | ✓ BCNF |
| order_item | (order_id, ISBN) | (order_id, ISBN) | ✓ BCNF |

**说明**：
- customer表中，email也具有唯一性约束，可作为候选键
- 所有表都满足BCNF，无冗余更新异常

## 4. 函数依赖分析

```
customer_id → customer_name, address, phone, email, password_hash, created_at, updated_at

```
order_id → customer_id, order_date, status, total_amount

**候选键**：order_id

### 4.4 订单明细表函数依赖

```
(order_id, ISBN) → quantity, unit_price
```

**候选键**：(order_id, ISBN)

## 5. 关系代数表达式

### 5.1 查询某客户的所有订单

```
π_order_id, order_date, status, total_amount(
    σ_customer_id=123(order)
)
```

### 5.2 查询订单详情（含图书信息）

```
π_order_id, ISBN, title, quantity, unit_price, quantity*unit_price(
    order_item ⋈(order_item.ISBN = book.ISBN) book
)
```

### 5.3 查询某图书的销售统计

```
γ_ISBN; SUM(quantity)→total_sold, SUM(quantity*unit_price)→total_revenue(
    σ_ISBN='9787111234567'(order_item)
)
```

### 5.4 查询购买过某图书的客户列表

```
π_customer_id, customer_name(
    customer ⋈(customer.customer_id = order.customer_id)
    (order ⋈(order.order_id = order_item.order_id)
    σ_ISBN='9787111234567'(order_item))
)
```

## 6. 数据完整性约束

### 6.1 实体完整性
- 所有表都有主键约束
- 主键不允许NULL值

### 6.2 参照完整性
| 外键 | 源表 | 目标表 | 删除规则 | 更新规则 |
|------|------|--------|---------|---------|
| customer_id | order | customer | RESTRICT | CASCADE |
| order_id | order_item | order | CASCADE | CASCADE |
| ISBN | order_item | book | RESTRICT | CASCADE |

### 6.3 用户定义完整性
- customer.email：UNIQUE, NOT NULL
- customer.password_hash：NOT NULL
- book.price：CHECK (price >= 0)
- book.stock：CHECK (stock >= 0)
- order_item.quantity：CHECK (quantity > 0)
- order_item.unit_price：CHECK (unit_price >= 0)
- order.status：ENUM约束取值范围
