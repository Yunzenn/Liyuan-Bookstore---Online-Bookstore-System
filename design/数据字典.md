# 数据字典

## 1. 数据项

### 1.1 客户相关数据项

| 数据项名 | 含义 | 类型 | 长度 | 取值范围 | 约束 |
|----------|------|------|------|----------|------|
| customer_id | 客户唯一标识 | INT | - | 1~2147483647 | 主键，自增 |
| customer_name | 客户姓名 | VARCHAR | 50 | 非空 | NOT NULL |
| address | 客户地址 | VARCHAR | 200 | - | - |
| phone | 联系电话 | VARCHAR | 20 | 数字和连字符 | - |
| email | 电子邮箱 | VARCHAR | 100 | 有效邮箱格式 | UNIQUE |
| password_hash | 密码哈希值 | VARCHAR | 255 | 非空 | NOT NULL |
| created_at | 注册时间 | DATETIME | - | 系统当前时间 | DEFAULT CURRENT_TIMESTAMP |
| updated_at | 更新时间 | DATETIME | - | 系统当前时间 | ON UPDATE CURRENT_TIMESTAMP |

### 1.2 图书相关数据项

| 数据项名 | 含义 | 类型 | 长度 | 取值范围 | 约束 |
|----------|------|------|------|----------|------|
| ISBN | 国际标准书号 | VARCHAR | 13 | 10位或13位数字 | 主键 |
| title | 书名 | VARCHAR | 200 | 非空 | NOT NULL |
| author | 作者 | VARCHAR | 100 | 非空 | NOT NULL |
| price | 定价 | DECIMAL | (10,2) | ≥ 0 | NOT NULL |
| publisher | 出版社 | VARCHAR | 100 | - | - |
| publish_year | 出版年份 | YEAR | - | 1900~当前年份 | - |
| stock | 库存数量 | INT | - | ≥ 0 | DEFAULT 0 |

### 1.3 订单相关数据项

| 数据项名 | 含义 | 类型 | 长度 | 取值范围 | 约束 |
|----------|------|------|------|----------|------|
| order_id | 订单唯一标识 | INT | - | 1~2147483647 | 主键，自增 |
| customer_id | 下单客户ID | INT | - | 存在于客户表 | 外键→customer |
| order_date | 下单时间 | DATETIME | - | 系统当前时间 | DEFAULT CURRENT_TIMESTAMP |
| status | 订单状态 | ENUM | - | 待付款/已付款/已发货/已完成/已取消 | DEFAULT '待付款' |
| total_amount | 订单总额 | DECIMAL | (10,2) | ≥ 0 | - |

### 1.4 订单明细相关数据项

| 数据项名 | 含义 | 类型 | 长度 | 取值范围 | 约束 |
|----------|------|------|------|----------|------|
| order_id | 所属订单ID | INT | - | 存在于订单表 | 联合主键，外键→order |
| ISBN | 图书ISBN | VARCHAR | 13 | 存在于图书表 | 联合主键，外键→book |
| quantity | 购买数量 | INT | - | ≥ 1 | NOT NULL |
| unit_price | 下单时单价 | DECIMAL | (10,2) | ≥ 0 | NOT NULL |

## 2. 数据结构

### 2.1 客户信息
```
客户信息 = 客户ID + 客户姓名 + 地址 + 电话 + 邮箱 + 密码哈希 + 注册时间 + 更新时间
```

### 2.2 图书信息
```
图书信息 = ISBN + 书名 + 作者 + 定价 + 出版社 + 出版年 + 库存数量
```

### 2.3 订单信息
```
订单信息 = 订单ID + 客户ID + 下单时间 + 订单状态 + 订单总额 + {订单明细}
订单明细 = ISBN + 书名 + 数量 + 单价 + 小计金额
```

### 2.4 客户注册请求
```
注册请求 = 客户姓名 + 地址 + 电话 + 邮箱 + 密码 + 确认密码
```

### 2.5 下单请求
```
下单请求 = 客户ID + {图书项}
图书项 = ISBN + 数量
```

## 3. 数据流

### 3.1 注册数据流
- **数据流名称**：客户注册请求
- **来源**：客户
- **去向**：验证信息处理
- **组成**：客户姓名 + 地址 + 电话 + 邮箱 + 密码
- **数据量**：约50-100条/天
- **峰值**：无明显峰值

### 3.2 登录数据流
- **数据流名称**：客户登录请求
- **来源**：客户
- **去向**：身份认证处理
- **组成**：邮箱 + 密码
- **数据量**：约200-500次/天

### 3.3 下单数据流
- **数据流名称**：客户下单请求
- **来源**：客户
- **去向**：创建订单处理
- **组成**：客户ID + {ISBN + 数量}
- **数据量**：约100-200单/天

### 3.4 订单查询数据流
- **数据流名称**：订单查询请求
- **来源**：客户/管理员
- **去向**：查询订单处理
- **组成**：订单号 / 时间范围 / 客户ID
- **数据量**：约500-1000次/天

## 4. 数据存储

### 4.1 客户表 (customer)
- **存储内容**：所有注册客户的基本信息
- **数据量估计**：初始1000条，年增长500条
- **存取频率**：读高写低（登录查询频繁，注册更新较少）
- **存取方式**：随机存取（按customer_id或email）

### 4.2 图书表 (book)
- **存储内容**：书店所有在售图书信息
- **数据量估计**：初始5000条，年增长1000条
- **存取频率**：读高写低（浏览查询频繁，上架更新较少）
- **存取方式**：随机存取（按ISBN）+ 顺序存取（列表浏览）

### 4.3 订单表 (order)
- **存储内容**：所有订单的头信息
- **数据量估计**：初始0条，年增长约50000条
- **存取频率**：写入频繁（每次下单），查询中等
- **存取方式**：随机存取（按order_id）+ 顺序存取（按时间查询）

### 4.4 订单明细表 (order_item)
- **存储内容**：订单中包含的具体图书和数量
- **数据量估计**：初始0条，年增长约100000条（平均每单2项）
- **存取频率**：与订单表同步
- **存取方式**：随机存取（按order_id + ISBN）

## 5. 处理过程

### 5.1 验证信息
- **处理名称**：验证客户注册信息
- **输入**：注册请求
- **输出**：验证结果（成功/失败原因）
- **处理逻辑**：
  1. 检查必填字段是否完整
  2. 验证邮箱格式是否正确
  3. 检查邮箱是否已被注册
  4. 验证密码强度
  5. 验证电话格式

### 5.2 身份认证
- **处理名称**：客户身份认证
- **输入**：登录请求（邮箱+密码）
- **输出**：认证结果（成功→客户信息 / 失败→错误提示）
- **处理逻辑**：
  1. 根据邮箱查询客户记录
  2. 验证密码哈希是否匹配
  3. 创建用户会话

### 5.3 创建订单
- **处理名称**：创建新订单
- **输入**：下单请求
- **输出**：订单创建结果（订单号/失败原因）
- **处理逻辑**：
  1. 验证客户是否存在且有效
  2. 验证所有图书ISBN是否存在
  3. 检查库存是否充足
  4. 计算每项小计和订单总额
  5. 生成订单记录和订单明细记录
  6. 扣减库存

### 5.4 查询订单
- **处理名称**：查询订单信息
- **输入**：查询条件
- **输出**：订单列表/详情
- **处理逻辑**：
  1. 根据查询条件构建SQL
  2. 关联查询订单表和订单明细表
  3. 关联查询图书信息获取书名
  4. 返回分页结果
