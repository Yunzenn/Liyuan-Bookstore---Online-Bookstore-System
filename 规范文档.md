# 网上书店数据库系统 - 规范文档

## 1. 命名规范

### 1.1 数据库命名规范

#### 数据库名
- 使用小写字母 + 下划线
- 示例：`online_bookstore`

#### 表名
- 使用小写字母 + 下划线，名词形式
- 使用单数形式（如 `customer` 而非 `customers`）
- 示例：`customer`、`book`、`order`、`order_item`

#### 字段名
- 使用小写字母 + 下划线
- 主键字段：`表名_id` 或有意义的名称（如 `ISBN`）
- 外键字段：与关联表主键同名
- 布尔字段：`is_` 前缀（如 `is_active`）
- 时间字段：`_at` 后缀（如 `created_at`、`updated_at`）
- 示例：`customer_name`、`order_date`、`unit_price`

#### 索引名
- 格式：`idx_表名_字段名`
- 唯一索引：`uk_表名_字段名`
- 示例：`idx_customer_email`、`uk_book_isbn`

### 1.2 Python代码命名规范

遵循 PEP 8 规范：

| 类型 | 规范 | 示例 |
|------|------|------|
| 模块名 | 小写 + 下划线 | `models.py`、`auth_routes.py` |
| 类名 | 大驼峰 | `Customer`、`BookInfo` |
| 函数/方法 | 小写 + 下划线 | `get_customer()`、`create_order()` |
| 变量 | 小写 + 下划线 | `customer_id`、`total_amount` |
| 常量 | 全大写 + 下划线 | `MAX_QUANTITY`、`DB_HOST` |
| 私有属性 | 单下划线前缀 | `_password_hash` |

### 1.3 前端命名规范

#### CSS类名
- 使用Bootstrap 5的工具类优先
- 自定义类：小写 + 连字符
- 示例：`book-card`、`order-detail`

#### JavaScript函数
- 小驼峰
- 示例：`addToCart()`、`submitOrder()`

#### 模板文件
- 小写 + 下划线
- 按模块分目录
- 示例：`templates/books/list.html`

## 2. SQL编写规范

### 2.1 基本规范

```sql
-- 关键字大写
SELECT customer_name, email
FROM customer
WHERE is_active = 1
ORDER BY created_at DESC;

-- 每个字段单独一行（复杂查询）
SELECT
    c.customer_id,
    c.customer_name,
    o.order_id,
    o.order_date,
    o.total_amount
FROM customer c
INNER JOIN order o ON c.customer_id = o.customer_id
WHERE o.status = '已完成'
ORDER BY o.order_date DESC;
```

### 2.2 建表规范

```sql
CREATE TABLE customer (
    customer_id     INT PRIMARY KEY AUTO_INCREMENT COMMENT '客户唯一标识',
    customer_name   VARCHAR(50) NOT NULL COMMENT '客户姓名',
    address         VARCHAR(200) COMMENT '地址',
    phone           VARCHAR(20) COMMENT '电话',
    email           VARCHAR(100) UNIQUE COMMENT '邮箱',
    password_hash   VARCHAR(255) NOT NULL COMMENT '密码哈希',
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='客户表';
```

### 2.3 注释规范
- 表级注释：说明表的用途
- 字段注释：说明字段含义、取值范围、约束
- 复杂查询：添加逻辑说明注释

## 4. 数据库设计规范

### 4.1 范式要求
- 所有表至少满足 **第三范式 (3NF)**
- 主键唯一标识每条记录
- 消除部分依赖和传递依赖

### 4.2 主键设计
- 优先使用自增整数作为代理主键
- 有业务意义的字段（如ISBN）可作为主键
- 避免使用复合主键（除关联表外）

### 4.3 外键设计
- 必须建立外键约束
- 级联策略：
  - `ON DELETE RESTRICT`：默认，防止误删
  - `ON UPDATE CASCADE`：主键更新时同步

### 4.4 字段类型选择

| 场景 | 推荐类型 | 说明 |
|------|---------|------|
| 整数ID | INT / BIGINT | 根据数据量选择 |
| 字符串 | VARCHAR(N) | 明确长度上限 |
| 金额 | DECIMAL(10,2) | 避免浮点精度问题 |
| 时间 | DATETIME | 包含日期和时间 |
| 枚举 | ENUM | 状态字段使用 |
| 布尔 | TINYINT(1) | 0/1表示 |

### 4.5 索引设计
- 主键自动建立聚簇索引
- 外键字段建立索引
- 常用查询条件字段建立索引
- 避免过度索引（影响写入性能）

## 5. 接口设计规范

### 5.1 URL设计

```
# 资源命名使用复数名词
GET    /api/books           # 获取图书列表
GET    /api/books/{isbn}    # 获取单本图书
POST   /api/orders          # 创建订单
GET    /api/orders/{id}     # 获取订单详情
PUT    /api/orders/{id}     # 更新订单
DELETE /api/orders/{id}     # 删除订单
```

### 5.2 响应格式

```json
// 成功响应
{
    "code": 200,
    "message": "success",
    "data": {
        "customer_id": 1,
        "customer_name": "张三"

// 错误响应
{
    "code": 400,
    "message": "参数错误",
    "errors": [
        "邮箱格式不正确"
    ]

// 列表响应
{
    "code": 200,
    "message": "success",
    "data": {
        "list": [...],
        "total": 100,
        "page": 1,
        "page_size": 10
    }

```

### 5.3 HTTP状态码使用

| 状态码 | 含义 | 使用场景 |
|--------|------|---------|
| 200 | OK | 成功 |
| 201 | Created | 创建成功 |
| 400 | Bad Request | 参数错误 |
| 401 | Unauthorized | 未登录 |
| 403 | Forbidden | 无权限 |
| 404 | Not Found | 资源不存在 |
| 500 | Internal Server Error | 服务器错误 |

## 6. 安全规范

### 6.1 密码安全
- 密码必须加密存储（使用 werkzeug.security 的 `generate_password_hash`）
- 密码强度要求：至少8位，包含字母和数字

### 6.2 输入验证
- 所有用户输入必须验证
- 验证规则：
  - 邮箱格式
  - 手机号格式
  - 数量为正整数
  - 金额为正数

### 6.3 会话管理
- 使用Flask内置Session
- 设置合理的超时时间
- 敏感操作需要重新验证
