## 3. 项目背景

实体书店”梨园书屋”运营中出现以下问题：
- 大量客户通过电话下订单
- 订单、出货、货款交付状态均以手工方式处理
- 管理效率低、易出错

**目标**：创建Web站点，实现订单、出货、货款交付状态的在线管理。

---

## 4. 功能需求

### 4.1 客户管理
| 需求项 | 详细说明 | 优先级 |
|--------|---------|--------|
| 客户注册 | 新客户首次登录前需申请账号，填写：客户名、地址、电话 | P0 |
| 唯一标识 | 系统为每个客户分配唯一标识号（customer_id） | P0 |
| 客户登录 | 使用邮箱/电话+密码登录 | P0 |
| 信息修改 | 客户可修改个人信息 | P1 |

### 4.2 图书管理
| 需求项 | 详细说明 | 优先级 |
|--------|---------|--------|
| 图书目录 | 包含：ISBN号、书名、作者、定价、出版年 | P0 |
| 浏览功能 | 客户可浏览图书目录 | P0 |
| 搜索功能 | 按书名、作者、ISBN搜索 | P1 |
| 库存管理 | 管理员可增删改图书信息和库存 | P1 |

### 4.3 订单管理
| 需求项 | 详细说明 | 优先级 |
|--------|---------|--------|
| 下订单 | 客户可对需要的书下订单 | P0 |
| 多订单 | 一个客户可以下多个订单 | P0 |
| 多种类 | 一个订单可包含多种不同图书（如：数据库教材30本+操作系统教材28本） | P0 |
| 订单状态 | 跟踪：待付款→已付款→已发货→已完成/已取消 | P0 |
| 出货管理 | 管理员可更新出货状态 | P0 |
| 货款状态 | 管理员可更新付款状态 | P0 |
| 订单查询 | 客户可查询自己的订单历史 | P0 |

## 5. 业务流程

### 5.1 客户注册流程
```
客户访问网站 → 点击注册 → 填写信息 → 验证邮箱唯一 → 生成customer_id → 注册成功
```

### 5.2 购物流程
```
登录 → 浏览图书 → 选择图书+数量 → 加入订单 → 确认订单 → 生成订单号 → 等待付款
```

### 5.3 订单状态流转
```
待付款 ──付款──▶ 已付款 ──发货──▶ 已发货 ──确认──▶ 已完成
   │                                              │
   └────────────────取消──────────────────────────┘
```

## 6. 数据需求

### 6.1 核心数据实体
| 实体 | 关键属性 | 说明 |
|------|---------|------|
| 客户 | customer_id, 客户名, 地址, 电话 | 注册用户 |
| 图书 | ISBN, 书名, 作者, 定价, 出版年 | 图书目录 |
| 订单 | order_id, 客户ID, 下单时间, 状态, 总额 | 订单头 |
| 订单明细 | order_id, ISBN, 数量, 单价 | 订单项 |

### 6.2 数据量预估
- 客户数：初期1000+，年增长500
- 图书数：初期5000+，年增长1000
- 订单数：年约50000单
